I

Bones

layout, responsiveness, IA, sim correctness — the broken things
01 Mobile / tablet are unusable bug

now At 375px the canvas shrinks to a ~60×35px thumbnail floating in dead space; the 300px-fixed rail dominates. Tablet (768px) crammed to ~470×260 with massive vertical waste. app.js:116-141

fix Stack vertically below ~1024px (canvas on top, rail below). Below ~640px collapse rail to a thin status strip + drawer for the rest.

pros

  • The world is the point — must be visible on every device
  • Reading the chronicle on a phone becomes pleasant
  • Forces us to decide which rail items are essential

cons

  • Touch interactions for the Hall / dev tools need design
  • Stacking breaks the "ambient dashboard" feel of desktop
  • Server is internal — is mobile actually a real audience?
decision Fix mobile + tablet. Cool to share with friends. Desktop first, then tablet, then mobile — accept that the desktop "ambient dashboard" feeling won't fully survive the stack.
02 Nav lives in the wrong place bug

now Triggers for /engine, /lab, and dev tools are crammed into the header row of TopColony — they have nothing to do with colonies. Three 27px glyphs, tooltip-only. app.js:22-26

fix Move to StatusLeft (top strip) as small labeled chips: "engine · lab · tools". Treat them as page-level nav, not section-level.

pros

  • Nav becomes discoverable without hover
  • TopColony recovers its header for actual colony context
  • StatusLeft is currently empty on the right side — wasted space

cons

  • StatusLeft becomes denser — voice shifts toward "dashboard"
  • Labels eat width — pixel-art glyph charm reduces
  • Need to decide if dev tools belongs in nav at all (it's dev-only)
decision Move all three to StatusLeft as labeled chips, including dev tools. Keep the gear visible — don't hide it behind ⌘. alone.
03 Rail names disconnected from the field bug

now Rail lists Cobblespore, Snufflebath, Bramblewort. The canvas shows three indistinguishable patches. No marker tells you which patch is which colony.

fix Hover the rail entry → soft outline blooms around that colony on the canvas. F57 label appears with the bloom. No permanent labels — preserve the diorama.

pros

  • The rail finally points at something
  • Watching becomes "I'm watching Cobblespore" not "I'm watching colonies"
  • Named entities feel alive when they have a location

cons

  • Labels burned into the world breaks the diorama illusion
  • Hover-bloom only works on desktop with a mouse
  • Pixel-art rendering of names is non-trivial — F57 font but kerning, occlusion
decision Hover-bloom approach. Names appear on demand, not burned in. See #11 — the names themselves also need to change to land properly.
04 Hyphae grow as a mat, not a network reframed · was polish bug

now At 21,408 hyphae across 3 colonies, the soil is a uniform yellow mat. This is the sim, not the renderer. Real hyphae should grow like roots — branching, exploring, leaving space. After the 2026-05-17 "hyphae-as-network-spine" change (cells persist on spent substrate), turnover + thickness rules don't keep up: tips keep extending into still-fertile cells while old cells refuse to leave.

fix Sim-side, in app/lib/sim.js. Candidate levers: tighten THICKNESS_MAX (fewer crowded cells allowed), raise TURNOVER_DIE_RISK past HYPHA_AGE_LIMIT, add tip "wandering" bias (extend away from filled neighbors, not into them), or reduce TIP_BIFURCATION_PROB when surrounded. The renderer halo + density curve from the original polish ticket becomes Plan B if sim tuning isn't enough.

pros

  • The world becomes readable again — individual colonies, individual mushrooms
  • Hyphae look like what they are: networks, not lawns
  • Mushroom fruitings become events you notice
  • Aligns sim with the project's own stated voice (root-like, ancient)

cons

  • Sim tuning is iterative — needs `/lab` runs to validate
  • Risk of swinging too far the other way (anemic, dying-back colonies)
  • BALANCE.md and engine field guide need updates after
  • Existing world.json may look strange after the rebalance
decision Sim-side fix is the priority. Investigate root cause in growth/turnover balance. Use /lab to validate. Renderer halo only if sim alone doesn't restore legibility.
II

Kit discipline

CLAUDE.md violations · debt that grows quietly
05 ~58 hardcoded hex strings, despite COL tokens existing debt

now overlays.jsx (33), shell.jsx (9), app.js (8), chronicle.js (6), primitives.jsx (2). Worst case: overlays + chronicle define aliases (H_TITLE, _CH_TITLE) for tokens that already exist in COL. CLAUDE.md forbids this. Stat primitive (a kit primitive) violates its own kit.

fix Sweep all hex → COL; delete H_* and _CH_* aliases; replace local font strings with MONO/SERIF/SERIF_BODY.

pros

  • One palette to tune; theme work becomes possible
  • Stops new code from copying the violations
  • Removes hidden visual drift between sibling components

cons

  • Mechanical, low-thrill work
  • Some hexes (#15110a, #5a4632) have no token — we'd add new tokens or accept "close enough"
  • Touches every kit file — risk of silent regressions if not eyeballed
decision Sweep in one focused PR. Screenshots before/after at desktop + tablet. Add 1-2 missing tokens rather than approximate.
06 TICKS_PER_DAY declared in 5 places arch

now Canonical in app/lib/time.js. Re-declared in app.js, canvas.js, chronicle.js, shell.jsx. No bundler so they can't import — but four of them silently rot if tick rate changes.

fix Ship ticksPerDay in /api/world/snapshot meta, OR add a tiny app/public/constants.js loaded first.

pros

  • Single source of truth
  • Tuning the tick rate stops being a bug-spawning event
  • Tiny change, big resilience win

cons

  • Snapshot payload grows slightly (negligible)
  • constants.js adds one more load-order dependency
  • Today there's no real bug — only hypothetical drift
decision Cheapest fix: snapshot.meta.ticksPerDay. Do it next time we touch the snapshot shape — bundle it with another change, not a standalone PR.
III

Pixel art & voice

where the polish lives · most of the interesting decisions
07 Canvas is parsimonious on big screens polish

now ShroomCanvas has maxWidth: 1280px. On 1440 desktop ~160px of dead canvas margin; on 1920, ~640px. canvas.js:600

fix Remove maxWidth. Let the canvas scale up (pixelated rendering preserves crispness).

pros

  • The world claims its surface — feels generous instead of cropped
  • Mushrooms become more legible just by being bigger
  • One-line change

cons

  • Pixels become chunky at 4K — some viewers love it, some hate it
  • Sun bloom and vignette were tuned at 1280; may need re-tuning
  • Bigger pixels can make the dense mat (see #04) look more overwhelming
decision Do it after #04. Sim density first, surface second. Re-tune bloom and vignette as part of the change.
08 TopColony tells you almost nothing polish

now Three rows of "1. Name · hyphae count". Compare to the richness in HallDetail (cap shape, color, age, phenotype). The living colonies get less information than the dead ones.

fix Add: tiny cap-shape preview (reuse HallMushroom at size=24), age in days. Skip trend arrows — they pull toward dashboard.

pros

  • Living colonies become characters, not labels
  • The cap-shape preview connects rail to canvas visually
  • Witness, not analyst — restraint preserved

cons

  • Rail gets denser — Chronicle has less vertical room
  • Cap-shape preview at 24px may not be legible enough
decision Cap-shape preview + age in days. Skip trend arrows. Keep the rail in service of "witness," not "analyst."
09 Loading state is off-voice polish

now "awakening…" in dim mono on a black page. The voice is "spiritual but not vague" — this is just a spinner without the spinner.

fix Italic serif passage ("the world stirs.") + optional pixel-noise bloom that resolves into the scene as snapshot arrives.

pros

  • First impression matches the voice everywhere else
  • Loading becomes part of the world, not a UI mode
  • Small scope — one component

cons

  • Loading is rare (server stays up) — diminishing return
  • Animated reveal needs care not to feel theatrical
decision Italic serif rewrite first. Animation only if it fits without feeling theatrical.
10 F57 / F35 bitmap fonts are underused polish

now The pixel fonts in tokens.jsx are clean and well-formed but only render inside tiny glyphs (engine/lab/dev triggers, hall mark). All visible reading text is web fonts. The bitmap fonts are committed but mostly silent.

fix Use F57 for in-world labels (the hover bloom from #03, "day 133" burned into a corner sigil). Use F35 for micro-stats inside pixel chips. Don't replace serif/mono — augment.

pros

  • The bitmap fonts become load-bearing instead of decorative
  • In-world text reads as part of the world, not a chrome overlay
  • Opens a real pixel-typography vocabulary for future work

cons

  • F57 has a small glyph set — anything not in [A-Z 0-9 . , : · - / ' " ! ? & + * ( ) ] renders as "?"
  • Pixel text doesn't scale — fixed visual size only
  • Easy to overdo and end up looking like a Game Boy menu
decision Tie directly to #03. The hover-bloom label is the natural first use. F35 micro-stats reserved for a later round.
11 Colony names are off-voice new voice

now The generator gives Cobblespore, Snufflebath, Bramblewort. The register is wrong — too whimsical, too Harry Potter. These read as cute, not strange-and-alive.

fix Rewrite the name generator with the register of real psilocybin cultivars: place-names (Mazatapec, Burma, Treasure Coast), physical-trait descriptors (Golden Teacher, Penis Envy), mystical-or-weird (Pearly Gates, Liberty Cap, Albino A+). All made up — no real species names. Aim for strange-but-believable.

pros

  • Names become part of the world's strangeness instead of softening it
  • Aligns with CLAUDE.md voice ("spiritual but not vague," "old-world without being affected")
  • Makes the hover-bloom (#03) actually land — a real name appearing in F57 is a moment; a whimsical one is wallpaper

cons

  • Risk of accidentally inventing a real cultivar name — needs a dedup check
  • "Penis Envy"-class names may feel jarring next to Urdu reverence — register needs care
  • The current names have shipped — long-time watchers may have grown attached
decision Rewrite the generator. Made-up names in the psilocybin-cultivar register. Bundle with #03 so the hover-bloom debuts with the new register, not the old one.
